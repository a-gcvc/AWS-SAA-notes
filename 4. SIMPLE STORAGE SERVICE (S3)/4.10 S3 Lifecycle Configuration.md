- **Transition Actions** change the storage class of whicever object or objects are affected.

- **Expiration Acttions** which can delete whatever object or objects versions are affected.

- Rules aren't based on access.

You can't move objects between classes based on how frequently they're accessed.

![alt text](<Screenshot 2025-08-20 151236.png>)

- S3 One Zone IA cannot transition into Glacier Instant Retrieval.

- Transition can't happen in an upward direction, only down.

- Object needs to be an S3 Standard for 30 days before it can be transitioned into Standard-IA or One Zone-IA

- If you want to create a single rule which transitions objects from Standard through to IA or One Zone-IA you have to wait additional 30 days before then transitioning those objects through to any of the Glacier classes.
