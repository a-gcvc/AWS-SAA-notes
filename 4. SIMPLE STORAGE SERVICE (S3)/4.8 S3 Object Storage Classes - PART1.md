- The default storage class available within S3 is known as **S3 Standard**.

- **S3 responds with a HTTP 1.1 200 OK status.**

- Makes data accessible immediately.

- When data is requested, it's available within milliseconds and objects can be made publicly available.

- **S3 Standard should be used for frequently accessed data, which is important and no replaceable.**

## S3 Standard-IA (infrequent access)

- Cheaper than S3 Standard.

- This class is cost-effective for data as long as you don't access the data very often, or you don't need to store it short term, or you don't store lots of tiny objects.

- **Should be used for long-lived data, but where data access is infrequent.**

- Don't use it for lost of small files, for temporary data or data which is constantly accessed and data which isn't important or which can be easily replaced because there's a better cheaper option for that.

![alt text](<Screenshot 2025-08-20 140244.png>)

## S3 One-Zone-IA

- Data stored using this class is only stored in one availability zone within the region.

- It doesn't have replication accross those additional AZs.

- Risk of data lose.

- Use for data which is non critical or data which can be easily replaced.
