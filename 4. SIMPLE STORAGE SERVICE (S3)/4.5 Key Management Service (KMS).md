- KMS provides **FIPS 40-2** compliant service - US security standard

![alt text](<Screenshot 2025-08-18 170252.png>)

- Nothing in KMS is ever stored in plaintext form persistently.

- KMS doesn't need to be told which KMS key to use for the decrypt operation. 

![alt text](<Screenshot 2025-08-18 171342.png>)

- KMS doesn't store the data encryption key in any way.

- KMS doens't do the encryption or decryption on data larger than 4KB using data encryption keys.

![alt text](<Screenshot 2025-08-18 171955.png>)

- By default, KMS keys are stored within the KMS service in that specific region.

They never leave region and they never leave the KMS service.

- Any interaction with a KMS key are done using the APIs available from KMS.

- **Rotation** is where physical backing material, so the data used to actually do cryptographic operations is changed.

With customer managed keys, rotation is optional.

With AWS managed keys, rotation can't be disabled.
