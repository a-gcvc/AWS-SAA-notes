- **Object versioning** is something which is controlled at a bucket level.

It starts off in a disabled state.

**Once enabled, you cannot disable it again.**

You can - **suspend it** and a suspended bucket can be re-enabled.

![alt text](<Screenshot 2025-08-18 154334.png>)

![alt text](<Screenshot 2025-08-18 154832.png>)

- When versioning on a bucket is disabled, the ID of the objects in that bucket are set to null.

- If you don't specify the ID, then it's assumed that you want to interact with the current version, the most recent version.

![alt text](<Screenshot 2025-08-18 155232.png>)

- **Delete marker** essentially is just a new version of that object.
S3 doesn't actually delete anything.

Delete marker makes it look deleted.

Delete marker is a special version of an object which hides all previous versions of that object.

You can delete Delete marker which essentially undelates the object, returning the current version to being active again.

- Even with versioning enabled, you can actually fully delete a version of an object, and that actually really deletes it.
