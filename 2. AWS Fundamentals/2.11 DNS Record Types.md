## MX records

- Your email server needs to know which server to pass the email onto.

- MX records have two main parts, a priority and a value. 

The value can be just a host. 

- The way that MX records are used is that our email server looks at the two address on the mail, and it focuses on the domain. (hi@google.com)

It then does and MX query using DNS (google.com), it looks to the root first, then dot com, then google.com, and then it retrieves any MX records.

- Lower values for the priority field are actually higher priority. 

- **MX record is how a server can find the mail server for a specific domain.**

- Allow you to add arbitrary text to a domain.

- Common usage: to prove domain ownership.

- It can be used to find spam.

## TTL 

- A TTL value is something that can be set on DNS records.

It's numeric value in seconds.

- Low values means more queries against your nameservers.
High values means less queries, but also less control if you need to change records.
