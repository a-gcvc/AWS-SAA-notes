## Subnets are what services run from inside VPCs

With AWS diagrams, **blue** means private subnets, **green** means public subnets.

# What is subnet?
- AZ resilient feature of a VPC
- Subnetnetwork of the VPC
- Part of VPC that is inside a specific availability zone
- Created in one availability zone and can never be changed. 
- If AZ fails, Subnet fails itself.
- Subnet can never be in multiple availability zones. 
- One AZ can have one or lots of subnets.
- IPv4 by default
- Allocated an IPv4 CIDR
- The CIDR that a subnet uses cannot overlap with any subnets in that VPC
- Optional can be created using IPv6 CIDR as long as the VPC also is enabled for IPv6
- Subnets can communicate with other subnets in the VPC

## Subnet IP Addressing
- 5 IP addresses within every VPC subnet that can't be used.
- **First address** is **newtork** address
- **Network + 1** VPC router (logical network device which moves data between subnets and in and out of the VPC if it's configured to allow that)
- **Network + 2** used for DNS
- **Network + 3** doesn't have a use yet; reserved for future requirements
- **Broadcast address** last IP in a subnet (Broadcaster not supported in VPC)

**DHCP Options Set** (Dynamic Host Configuration Protocol) - how computing devices receive IP addresses automatically.
