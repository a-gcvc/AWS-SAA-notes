Security groups are **stateful**, they detect response traffic automatically for a given request. 

Don't need to worry about configuring ephemeral ports, it's all handled by the product. 

Limitation - **no explixit deny** (can't block specific bad *actors*)

Operate above NACLs on the OSI 7 layer stack - have more features.

They support IP and CIDR based rules and also allow reference AWS logical resoursces. 

Security groups are not attached to instances, nor are they attached to subnets. 

They are attached to specific elastic network interfaces known as ENIs. 

# Logical References 
When you reference a security group from another security group, you're actually referencing any resources which have that security group associated with them. 

# Self References
Allowing all traffic.
If it's attached to all of the instances, then anything with this security group is attached can receive communications, so all traffic from the security group. 

It allows communicattions to occur to instances which have it attached from instances which have it attached. 

It handles any IP changes automatically which is useful it these instances are within an auto scaling group which is provisioning and terminating instances based on load on the system.
