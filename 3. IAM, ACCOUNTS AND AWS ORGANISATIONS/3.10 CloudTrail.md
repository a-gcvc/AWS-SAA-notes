- **CloudTrail** Is a product which logs API calls and account events.

- **CloudTrail event** is a record of an activity in an a AWS account.

- CloudTrail is regional service.

![alt text](<Screenshot 2025-08-18 121748.png>)

- **Management events** provide information about management operations that are performed on resources in your AWS account. (control plane operations)

By default, CloudTrail only logs Management events because **data events** are often higher volume.

- A **CloudTrail trail** is the unit of configuration within the CloudTrail product.

It's a way you provide configuration to CloudTrail on how tto operate.

**A trail logs events for the AWS region that it's created in.**

![alt text](<Screenshot 2025-08-18 125036.png>)

- **Single-region trail** is only ever in the region that it's created in, and it only logs events for that region.

- **All-region-trail** you can think of as a collection of trails in every AWS region, but it's managed as one logical trail.

- Most services log events in the region where the event occured.

A very small number of services log events globally to one region. (us east 1)

- Data events is not something that's generally enabled by default. 

It's something you have to explicitly set when you create a trail.

- **Organization trail**: if you create this trail from the management account of an organization, it can store all of the information for all of the accounts inside that organization.

- CloudTrail typically delivers log files within 15 minutes of the account activity occuring.
